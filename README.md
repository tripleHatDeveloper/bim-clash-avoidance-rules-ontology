# BIM Coordination Rules and Ontology for Clash Avoidance

## Overview

This repository provides a structured dataset and ontology supporting clash avoidance in BIM-based multidisciplinary coordination. It includes a validated set of coordination rules and an OWL ontology for formalising and evaluating coordination knowledge.

---

## Contents

### /rules  
A curated set of 35 coordination rules derived from expert input.  

Each rule is structured using:
- **Condition** – the context in which the rule applies  
- **Constraint** – the requirement to be satisfied  
- **Rationale** – the coordination issue the rule aims to prevent  
- **Classification** – e.g. mandatory, recommendation, or best practice  

### /ontology  
An OWL ontology representing coordination concepts, relationships, and constraints.  

It supports:
- Formal representation of coordination knowledge  
- Logical consistency checking  
- Extensibility for additional rules and domains  

---

## Scope

The dataset focuses on coordination practices related to architectural, structural, and MEP disciplines, with emphasis on preventing hard clashes during model authoring.

---

## Usage

The materials can be used for:
- Rule-based checking in BIM workflows  
- Research on coordination knowledge formalisation  
- Development of decision-support or automation tools in BIM environments  

---

## Notes

- The dataset is independent of any specific BIM software implementation  
- No proprietary project data or BIM models are included  
- The rules reflect expert-informed coordination practices and may require adaptation to specific project contexts
